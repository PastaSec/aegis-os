AEGIS OS WHITEPAPER
Version 0.2 Draft
Project: PastaSec/aegis-os

Title:
AEGIS OS — Offline-First Field Terminal for Local Resilience

Mission:
AEGIS OS exists to help individuals and neighborhoods remain informed, connected, and capable every day, and especially when normal infrastructure is unavailable.

Executive Summary:
AEGIS OS is an offline-first operating environment designed for Raspberry Pi-based field terminals. It combines local knowledge storage, field journaling, inventory tracking, hardware monitoring, and future communications tools into a single appliance-style interface.

AEGIS is not intended to be a general-purpose Linux desktop. It is designed to feel like dedicated equipment: simple, readable, reliable, and useful under stress.

Core Principle:
If the Internet disappeared tomorrow, AEGIS should become more valuable, not less.

System Identity:
AEGIS stands for:

Autonomous Emergency & General Information System

Its core values are:

Knowledge.
Communication.
Resilience.

Stronger Together. Prepared Together.

Design Philosophy:
AEGIS OS is built around five rules:

1. Offline First
Critical functionality must not depend on Internet access.

2. Three Keystrokes Maximum
Important information should be reachable quickly.

3. No Linux Exposure
The operator should not need to understand Python, virtual environments, YAML, Markdown, systemd, or Linux internals during normal use.

4. Everything Searchable
Knowledge, journal entries, inventory, hardware status, and future communications should all be discoverable through one search interface.

5. Consistent Navigation
Every screen should use the same interaction model:
Up/Down to move, Enter to open, Esc to return, / to search, R to refresh, Q to standby.

Current Capabilities:
The current alpha build includes:

- AEGIS logo boot splash
- Appliance-style loading sequence
- PiTFT-optimized interface
- Mission dashboard
- Knowledge pack browser
- Markdown document viewer
- Field Journal foundation
- Inventory foundation
- Universal search foundation
- GitHub-backed source control
- Plain-file data architecture

Field Terminal Direction:
Version 0.2 moves AEGIS from a prototype dashboard into a field terminal.

The home screen will focus on mission areas:

- Knowledge
- Journal
- Inventory
- Communications
- Navigation
- Hardware
- System

Technical diagnostics such as CPU, memory, storage, temperature, and power status will move into Hardware. Configuration, logs, updates, backup, restore, and about information will move into System.

Knowledge Architecture:
AEGIS uses Knowledge Packs rather than hardcoded content.

A Knowledge Pack is a portable offline information bundle containing a manifest and human-readable documents. Packs may include Markdown, text, PDF, HTML, images, and future formats such as ZIM archives.

This allows communities to create and share localized packs such as:

- Florida Hurricane Pack
- CERT Pack
- First Aid Pack
- Ham Radio Pack
- Linux Field Reference
- Generator Safety Pack
- Community Resource Pack

Plain-File Architecture:
AEGIS intentionally stores core data as ordinary files.

Knowledge lives in:

knowledge/packs/

Journal entries live in:

journal/

Inventory lives in:

inventory/

This makes the system portable, inspectable, editable, backup-friendly, and resilient. The data remains useful even without the AEGIS application.

- Meshtastic
- LoRa
- APRS
- NOAA weather radio references
- GMRS/FRS references

- Waypoints

- Grid coordinates

Hardware Integration:
AEGIS is intended to support modular hardware integrations, including:

- ESP32 sensors
- Battery monitors
- GPS receivers
- LoRa radios
- SDR devices
- USB storage
- Cameras
- Flipper Zero companion workflows
- Environmental sensors

Community Vision:
AEGIS OS is designed for individuals first, but neighborhoods next.

A future AEGIS node may serve local knowledge, bulletins, contacts, and resources over a local Wi-Fi network without Internet access. Multiple AEGIS nodes may eventually synchronize selected community information.

AEGIS is not a prepper gadget. It is a practical resilience terminal useful on ordinary days and more valuable during disruptions.

Conclusion:
AEGIS OS is evolving into an offline-first field terminal for local resilience.

Its purpose is simple:

Preserve knowledge.
Strengthen communities.
Remain operational.
