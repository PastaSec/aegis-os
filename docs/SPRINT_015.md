# Sprint Goal

Introduce a lightweight Knowledge Pack index generated by AEGIS Foundry and consumed by the AEGIS Runtime.

The Runtime should be able to discover large Knowledge Packs without parsing every Markdown document during startup.

---

# Background

Knowledge Packs currently consist of:

manifest.yaml

plus Markdown documents under:

docs/

The Runtime recursively loads every Markdown document to build its internal document list.

This approach is simple and reliable but becomes increasingly expensive as Knowledge Packs grow.

Foundry already performs document import, normalization, metadata generation, and validation.

It is therefore the correct place to generate an optimized index for Runtime consumption.

---

# Objectives

Introduce an optional:

index.json

for every Knowledge Pack.

Move metadata indexing into Foundry.

Allow Runtime to consume the generated index.

Preserve compatibility with existing Knowledge Packs.

---

# Knowledge Pack Layout

A production Knowledge Pack may now contain:

```text
knowledge/
└── packs/
    └── fema/
        ├── manifest.yaml
        ├── index.json
        └── docs/
            ├── preparedness/
            ├── power/
            └── weather/
```

The index is generated automatically.

Operators should never edit it manually.

---

# Index Contents

Each indexed document should include lightweight metadata.

Suggested fields:

```json
{
  "title": "Emergency Supply Kit",
  "path": "docs/preparedness/emergency-supply-kit.md",
  "summary": "Essential emergency supplies for 72 hours.",
  "category": "Preparedness",
  "tags": [
    "preparedness",
    "supplies",
    "emergency"
  ],
  "author": "FEMA",
  "revision": "2025",
  "size": 2418
}
```

Document bodies remain in Markdown.

The index exists solely to improve discovery.

---

# Runtime Behavior

When loading a Knowledge Pack:

1. Read manifest.yaml.
2. If index.json exists:
   - Load document metadata from the index.
3. Otherwise:
   - Fall back to the current recursive Markdown discovery.
4. Open Markdown only when the operator selects a document.

The operator should notice no behavioral difference.

Only startup performance and scalability improve.

---

# Foundry Behavior

Foundry should generate index.json whenever a Knowledge Pack is built or imported.

Validation should verify:

- index.json exists
- every indexed document exists
- every Markdown document is represented
- metadata matches the source document

Validation failures should remain human-readable.
