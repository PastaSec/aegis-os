# Executive Summary

AEGIS OS is an offline-first field operating system designed to preserve knowledge, strengthen communities, and remain operational when traditional infrastructure is unavailable.

AEGIS is not a Linux distribution.

AEGIS is not a dashboard.

AEGIS is not a web application.

AEGIS is a dedicated field terminal designed to operate on inexpensive hardware while providing immediate access to critical information.

The system should feel more like operating professional equipment than using a computer.

Inspirations include:

- Garmin GPS receivers
- Aircraft multifunction displays
- Public safety terminals
- Emergency Operations Center consoles
- Amateur radio field equipment
- Military field computers

The underlying operating system is an implementation detail.

The operator should simply know they are using **AEGIS OS**.

---

# Mission

Preserve knowledge.

Strengthen communities.

Remain operational.

AEGIS exists to make useful information available regardless of Internet connectivity.

Knowledge should never become inaccessible because a service disappears, a website goes offline, or communications fail.

---

# Vision

Every individual, family, volunteer organization, CERT team, amateur radio operator, and community should be able to deploy an inexpensive field terminal containing the knowledge they need to respond confidently to everyday situations and extraordinary emergencies.

AEGIS should become the definitive open-source offline resilience platform.

# Design Philosophy

AEGIS is an appliance.

The operator should never need to understand:

- Linux
- Python
- YAML
- Markdown
- systemd
- package managers

Those are implementation details.

The experience should resemble turning on a dedicated device.

Power.

Boot.

Mission Ready.

---

# Primary Capabilities

AEGIS revolves around capabilities rather than applications.

Current capabilities include:

- Knowledge
- Journal
- Inventory
- Hardware
- System

Future capabilities include:

- Communications
- Navigation
- Weather
- GPS
- Offline Maps
- Community
- SDR

# Core Principles

## Offline First

Critical functionality must never depend on Internet connectivity.

Internet access enhances AEGIS.

It never defines AEGIS.

---

## Human First

Interfaces should reduce stress.

Never increase it.

Navigation must be obvious.

Information must be readable.

Interaction should remain consistent everywhere.

---

## Plain Files

Knowledge belongs to the user.

Store information using ordinary files whenever practical.

Examples:

Markdown

Text

Images

PDF

JSON

Future-proof data is more valuable than clever databases.

---

## Keyboard First

Every feature must function with:

Arrow Keys

Enter

Escape

Slash Search

No mouse should ever be required.

---

## Three Keystrokes Maximum

Important information should never be more than three navigation steps away.

If additional navigation becomes necessary, redesign the interface.

---

## Everything Searchable

Universal search is a core capability.

Search should include:

Knowledge

Journal

Inventory

Hardware

Future communications

Future maps

Future plugins

One search.

Everything.

Documents should remain readable without AEGIS.

Knowledge Packs should remain portable.

Examples:

Florida Hurricane Pack

CERT Pack

Medical Pack

Generator Pack

Ham Radio Pack

Community Resource Pack

---

# Journal Philosophy

The journal is a field notebook.

Not a word processor.

Entries should be:

Fast

Timestamped

Searchable

Portable

Human-readable

---

# Inventory Philosophy

Inventory exists to answer:

What do I have?

Where is it?

How much remains?

When does it expire?

Nothing more complicated than necessary.

---

# Hardware Philosophy

Hardware status belongs in Hardware.

Never clutter the home screen with diagnostics.

The home screen should answer only one question:

"What would you like to do?"

---

# Plugin Philosophy

Every major capability should become a plugin.

Plugins should be independent.

Installing a plugin should never require changing the core.

---

# Engineering Principles

Main always runs.

Maintainability

Reliability
